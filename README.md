# Galactic Ledger
「情報の可視化」「最適化」「市場操作」を楽しむMMOゲーム

## 1. ゲームのコアコンセプト

派手な3Dアクションではなく、「情報の可視化」「最適化」「市場操作」を楽しむゲームです。<br>
プレイヤーの役割はパイロットではなく、星間企業のCEOです。

## 2. 4つの柱の詳細仕様

### A. 土地：惑星と「ステーション」

#### スロット制の土地開発
  - 惑星はグラフィックではなく、データとして扱います。
  - 各惑星には「建築スロット」があります。
  - プレイヤーは土地に入植し、「採掘施設」「精製工場」「居住区」などを建てます。
#### ハブ（宇宙ステーション）
  - 惑星とは別に、宇宙空間に「交易所（ステーション）」が存在します。ここが物流の拠点となります。
  - 上位プレイヤーは、自分のステーションを建設し、テナント料（着陸料）を徴収することができます。

### B. 資源：複雑なサプライチェーン

  - **ティア（階層）構造**
  - **Tier 1（一次資源）**
    - 水、鉄鉱石、ガスなど
    - 誰でも採掘可能
  - **Tier 2（加工品）**
    - 鉄板、燃料、酸素
    - Tier 1が必要
  - **Tier 3（ハイテク）**
    - CPU、エンジン、船体
    - 高度な工場が必要
  - **相互依存**
    - 初心者はTier 1を掘って稼ぎ、上級者はそれを買い取ってTier 3を作ります。これにより、初心者と上級者の間に経済的な協力関係が強制的に生まれます。

### C. 輸送：このゲームの生命線

ブラウザゲームなので、自分で船を操縦するのではなく、「航路（ルート）の設定」がメインです。

#### 契約（コントラクト）システム
  - 「A星からB星へ、鉄鉱石1000個を運んでほしい。報酬は500クレジット」という依頼（配送クエスト）をプレイヤー自身が発行します。
  - 輸送特化のプレイヤーは、これを受注して稼ぎます。

#### 物流コストの概念
  - 燃料代と時間を計算し、利益が出るギリギリのラインを見極める必要があります。
  - 遠くの星なら高く売れますが、輸送に時間がかかり（リアルタイムで数時間～数日）、燃料代もかかります。

### D. 外交：市場と企業同盟

#### ローカル市場 vs グローバル市場
  - 魔法のような「全宇宙共通の倉庫」はありません。**物は物理的にそこになければ売れません。**
  - 「地球周辺では鉄が暴落しているが、辺境のコロニーでは鉄が高騰している」といった地域差が生まれ、これが商機となります。

#### 企業（ギルド）
  - 株式発行システム：プレイヤー企業は株式を発行して資金調達ができます。
  - 配当金：利益が出たら株主（他プレイヤー）に配当を出します。
  - 経済制裁：敵対企業の製品をボイコットしたり、特定航路を封鎖するなどの圧力をかけられます。

## 3. 「終わりのない」ための経済バランス（インフレ対策）

経済シミュレーションで最も恐ろしいのは、ゲーム内通貨が増えすぎて価値がなくなることです。これを防ぐための**「資源の消費先（シンク）」**を設けます。

1. **維持費（Maintenance）**<br>
  建物や船は時間経過で劣化します。修理し続けないと機能停止します。
2. **燃料（Fuel）**<br>
  移動には必ず燃料（資源）を消費します。
3. **食料（Life Support）**<br>
  惑星の労働者（人口）は食料と水を消費し続けます。供給が止まると人口が減り、生産が止まります。
4. **贅沢品（Luxury）**<br>
  上位の効率を出すためには、労働者に嗜好品（ゲーム的な意味はないが高価な消費財）を与える必要があります。

### 4. ブラウザ画面のイメージ (UI/UX)

画面は「ダッシュボード」のような見た目になります。

  - **左パネル** <br>
      所有している惑星と艦隊のリスト。現在のステータス（移動中、生産中、待機中）。 
  - **中央メイン**
    - **星図モード** <br>
      ノード（星）とライン（航路）で描かれたネットワーク図。
    - **市場モード** <br>
      株取引ツールのような「板（オーダーブック）」情報。「買い注文」「売り注文」が並ぶ。
  - **右パネル** <br>
    通知（「船団Aが到着しました」「オークションで競り負けました」）。
