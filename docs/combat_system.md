# 戦闘システム (Combat System)

## 1. 基本コンセプト
本作の戦闘は、プレイヤーの反射神経を競うアクション (Twitch skill)ではなく、**マネジメント (準備と投資)**と**マクロな戦略**に焦点を当てています。
プレイヤーは現場のパイロットではなく、艦隊全体を指揮する提督や企業のCEOとして振る舞います。

### マクロRTS (司令官視点)
- **非同期プレイの重視**<br>
プレイヤーが24時間ログインしていなくても、事前に設定した「作戦 (ドクトリン)」に従って艦隊が自動で行動・防衛します。
- **スローペース**<br>
移動や戦闘の解決にはリアルタイムで数分〜数時間かかります。これにより、通知を見てから介入する余地が生まれます。

## 2. 戦闘メカニクス

### 3すくみ (Rock-Paper-Scissors)
経済を回すため、「最強の船」を作らせないバランス調整を行います。
- **エネルギー兵器 (レーザー)**<br>
シールドに強く、装甲に弱い。
- **実弾兵器 (レールガン)**<br>
装甲に強く、回避が高い船に当たらない。
- **爆発兵器 (ミサイル)**<br>
追尾して必中だが、迎撃システム (PD)に撃ち落とされる。

### インターディクション (妨害)
戦闘は「待ち伏せ」と「遭遇」によって発生します。
1. **隠密判定**<br>
輸送船の「隠密性」 > 敵の「スキャン強度」なら戦闘回避。
2. **逃走判定**<br>
見つかった場合、「エンジン出力」 > 敵の「妨害強度 (Warp Scrambler)」なら緊急離脱。
3. **戦闘開始**<br>
上記に失敗した場合、バトルフェーズに移行します。

### 指揮システム (ROE: Rules of Engagement)
ユニットを直接操作するのではなく、**「交戦規定 (ROE)」**を設定します。
- **積極攻撃 (Aggressive)**<br>
視界内の敵を自動追尾・攻撃する。
- **防衛維持 (Defensive)**<br>
指定座標から離れず、侵入者のみを攻撃する。
- **逃走優先 (Evasive)**<br>
敵を感知したら即座に逆方向へ逃げる。

## 3. RTS要素とマップ

### シームレスな宇宙空間
- ヘックスやグリッドを使わず、座標 (X, Y)で管理される広大な宇宙。
- **視界 (Fog of War)**<br>
レーダー範囲外は暗闇となり、偵察の価値が生まれます。

### 基地建設 (タワーディフェンス)
- 資源採掘地や工場の周辺に、物理的に「防衛プラットフォーム」や「シールド発生装置」を配置します。
- 敵の侵攻ルートを予測した配置 (死角を消すなど)が重要になります。

### ロジスティクス (補給線)
- 艦隊は「弾薬」と「燃料」を消費し続けます。
- 戦闘艦隊だけでなく、後方の「輸送船団」を守る必要があります。逆に、敵の補給線を断つことで無力化する戦略も有効です。

## 4. 経済へのフィードバック

戦闘結果は経済システムに直接的な影響を与えます。

### デブリ (残骸)回収
- 破壊された船からは、建造資源の一部 (例: 30%)と積荷が「デブリ」として宇宙空間に残ります。
- これを回収する「サルベージ船」が必要となり、戦闘後の新たな経済活動を生みます。

### 傭兵 (Mercenary)
- 戦闘力を持たない輸送専門プレイヤーは、システム上で「護衛依頼」を発行できます。
- 戦闘専門プレイヤー (傭兵)はこれを受注し、護衛料で稼げます。

### 保険システム
- 船が撃沈された際のリスクヘッジとして、事前に保険料を支払うことで船体価格の一部が補填されます。
